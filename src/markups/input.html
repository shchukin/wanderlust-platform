<textarea>

Don’t forget to use the rows attribute to specify number of lines.


input--expandable

Using JavaScript, calculate the height to make the textarea expand as the user types. In this case, the rows attribute will serve as a starting point, and the max-height set in the styles will limit the computed height, even if it exceeds any reasonable value.

The value for this max-height is calculated based on the number of lines plus vertical paddings and borders.




<select>





Placeholder in the <select>

Notice that there is a problem with the <select> tag. It doesn’t work with the placeholder attribute. It’s expected that at least one option is always selected.

However, it doesn’t match the rest of the inputs, especially when there are a few of them in a row along with a select. So let’s fake it.

Assign first option to be a placeholder: <option value="placeholder">. Then use JS to check if this option is selected and toggle the input--placeholder-is-chosen class to change the color.



Don’t make option with placeholder to be disabled. Users may want to leave this field untouched, or clear it by re-selecting placeholder. If the form was submitted with "placeholder" as the value for the select, it should be treated as equivalent to an empty text field.

On the other hand, if the field is required, ensure that it does not pass validation with the placeholder selected.
