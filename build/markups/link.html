.link--fake

For the case when we don’t need the functionality of <a>, but still must use it.

In certain cases browsers attempt to detect links where there are none. For example, long sequences of digits can be mistaken as a phone number. Some dates on the page can be interpreted as information suitable for adding to a calendar. Or the user’s email on hos own page, which shouldn’t be clickable, can still be seen as a link to an email.

In these situations, browsers (especially mobile once) often wrap it with an <a> tag. As a result default blue styling will be applied. To prevent this from happening, we can add the <a> manually remove its styling:
