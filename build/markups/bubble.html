First, the simplest case: bubble with a couple of words that would fit any screen an has no problems with line breaks or positioning:




Lorem ipsum dolor sit amet


Problems come when you need more text.

Since bubbles are positioned absolutely, the text in them collapses, trying to take up as little space as possible (same as min-content in the CSS). Line breaks would occur after every word.

Also, the browser can't analyze the text in any way to decide where to place line breaks or width (width is undefined in the bubble).

To avoid this, line breaks are disabled by default (using white-space: nowrap):




Lorem ipsum dolor sit amet, consectetur adipisicing elit. Aperiam asperiores, aspernatur blanditiis doloribus error excepturi omnis optio pariatur placeat velit.

If line breaks are still needed, you can either add them manually (using the <br> tag), which is suitable for small bubbles:






Lorem ipsum dolor sit amet,
consectetur adipisicing elit.
Animi aspernatur consequuntur.

Or you can write custom markup in the bubble: add additional components with a fixed width and disabled nowrap. In this case, you can display any text:







Lorem ipsum dolor sit amet, consectetur adipisicing elit. Atque eius in voluptatem. Consectetur culpa cupiditate deleniti illo in iste qui. Eos ex explicabo impedit, numquam officia provident sunt unde vitae!



Showing the Bubble

Actually, bubbles are hidden by default. In all the examples above, they are visible just for the convenience of reading this doc. But this is how they are being shown.

1) On touch devices we catch clicking on .bubble-handler, the script finds and shows the related bubble. The related bubble is the one located inside the same .bubble-context.

2) On desktops, the bubble is shown by simply hovering over .bubble-context via CSS.

Real example in action:

Lorem ipsum dolor sit amet consectetur adipisicing elit



Positioning and Screen Edge Clipping

There is a problem: bubble gets clipped if it's located too close to the screen edge. So there is a script running and checking the distance to the edge, and if the bubble is clipped, it shifts it in the opposite direction. Shrink the browser window and see it in the examples above (you can make bubble stay on the page clicking it once).
